# /db-list — Управление реестром баз данных

Управляет файлом `.v8-project.json` — локальным реестром информационных баз проекта. Файл хранит параметры подключения, алиасы, привязку к веткам Git и структуру исходников. Локальные секреты и машинные переопределения можно хранить в `.v8-project.local.json`.

Если проект только скачан из Git и `.v8-project.json` отсутствует, используй
`/db-list init`: он создаёт локальный конфиг из `.v8-project.json.example`
или по обнаруженной структуре каталогов.

## Usage

```
/db-list                    — показать список баз
/db-list init               — инициализировать .v8-project.json после clone
/db-list add                — добавить базу (интерактивно)
/db-list remove <id>        — удалить базу из реестра
/db-list show <id|alias>    — подробности по базе
```

## Формат `.v8-project.json`

Файл размещается в корне проекта (рядом с `.git/`).

```json
{
  "v8path": "C:\\Program Files\\1cv8\\8.3.25.1257\\bin",
  "configSrc": "cfsrc",
  "extensions": "extensions",
  "reports": "reports",
  "processing": "processing",
  "databases": [
    {
      "id": "dev",
      "name": "Разработка",
      "type": "file",
      "path": "C:\\Bases\\MyApp_Dev",
      "user": "Admin",
      "password": "",
      "aliases": ["dev", "разработка"],
      "branches": ["dev", "develop", "feature/*"]
    },
    {
      "id": "test",
      "name": "Тестовая",
      "type": "server",
      "server": "srv01",
      "ref": "MyApp_Test",
      "user": "Admin",
      "password": "",
      "aliases": ["test", "тест"]
    }
  ],
  "default": "dev"
}
```

## Шаблон `.v8-project.json.example`

Если `.v8-project.json` не хранится в репозитории, проект может хранить рядом
шаблон `.v8-project.json.example`. В шаблон можно включать структуру проекта и
пример баз без секретов:

```json
{
  "configSrc": "cfsrc",
  "extensions": "extensions",
  "reports": "reports",
  "processing": "processing",
  "databases": [
    {
      "id": "dev",
      "name": "Разработка",
      "type": "server",
      "server": "localhost",
      "ref": "MyApp_Dev",
      "user": "",
      "aliases": ["dev"]
    }
  ],
  "default": "dev"
}
```

При `/db-list init` сначала используй `.v8-project.json.example`, если он есть.
Если шаблона нет, определи структуру по каталогам проекта (`cfsrc`,
`extensions`, `reports`, `processing`) и спроси недостающие параметры базы.

## Локальные переопределения `.v8-project.local.json`

Если рядом с `.v8-project.json` есть файл `.v8-project.local.json`, прочитай
оба файла и применяй local-файл поверх основного. Local-файл предназначен для
паролей, локальных имён пользователей, путей к инструментам и других значений,
которые не должны попадать в git.

Рекомендуемый формат для секретов базы:

```json
{
  "databases": {
    "dev": {
      "user": "Admin",
      "password": "secret"
    }
  }
}
```

Правила применения:

- корневые поля local-файла (`v8path`, `webPath`, `ffmpegPath`, `tts` и т.д.)
  переопределяют одноимённые поля `.v8-project.json`;
- если `databases` в local-файле — объект, его ключи считаются `id` баз, а
  значения накладываются на соответствующие записи основного массива
  `databases[]`;
- если `databases` в local-файле — массив, записи сопоставляются по `id` и
  накладываются на записи основного массива;
- local-файл не должен создавать новые базы без основной записи в
  `.v8-project.json`: если база есть только в local-файле, считай это ошибкой
  конфигурации и попроси добавить базу через `/db-list add`;
- при выводе `/db-list show` не показывай пароль целиком; достаточно указать,
  что пароль задан в local-файле.

Рекомендуемые строки `.gitignore` проекта:

```gitignore
.v8-project.local.json
*.local.json
```

### Поля корневого объекта

| Поле | Тип | Описание |
|------|-----|----------|
| `v8path` | string | Каталог bin платформы 1С. Необязательный — если не задан, автоопределение |
| `databases` | array | Массив баз данных |
| `default` | string | id базы по умолчанию |
| `configSrc` | string | Каталог XML-выгрузки конфигурации по умолчанию |
| `extensions` | string | Каталог исходников расширений |
| `reports` | string | Каталог исходников внешних отчётов |
| `processing` | string | Каталог исходников внешних обработок |

### Поля объекта базы данных

| Поле | Тип | Обязательное | Описание |
|------|-----|:------------:|----------|
| `id` | string | да | Уникальный идентификатор (латиница, без пробелов) |
| `name` | string | да | Человекочитаемое имя |
| `type` | `"file"` / `"server"` | да | Тип подключения |
| `path` | string | для file | Путь к каталогу файловой базы |
| `server` | string | для server | Адрес сервера 1С |
| `ref` | string | для server | Имя базы на сервере |
| `user` | string | нет | Имя пользователя 1С |
| `password` | string | нет | Пароль; для секретов предпочитай `.v8-project.local.json` |
| `aliases` | string[] | нет | Альтернативные имена для быстрого доступа |
| `branches` | string[] | нет | Git-ветки или glob-паттерны (`release/*`, `feature/*`), привязанные к этой базе |
| `configSrc` | string | нет | Каталог XML-выгрузки конфигурации для этой базы; если не задан, используй корневой `configSrc` |

## Алгоритм разрешения базы данных

Этот алгоритм используется ВСЕМИ навыками (`db-*`, `epf-build`, `epf-dump`, `erf-build`, `erf-dump`) для определения целевой базы. Перед разрешением базы всегда применяй `.v8-project.local.json`, если он существует.

1. Если пользователь указал **параметры подключения** (путь, сервер) — используй напрямую
2. Если пользователь указал **базу по имени** — ищи совпадение в таком порядке:
   1. По `id` (точное совпадение)
   2. По `aliases` (совпадение в массиве с учётом морфологии: «тестовую» = «тестовая» = «тестовой»)
   3. По `name` (нечёткое совпадение с учётом морфологии и регистра)
3. Если пользователь **не указал** базу — сопоставь текущую ветку Git с `databases[].branches`:
   - Точное совпадение: ветка `dev` → `"branches": ["dev"]`
   - Glob-паттерн: ветка `release/2.1` → `"branches": ["release/*"]`
4. Если ветка не совпала — используй `default`
5. Если не найдено или неоднозначно — спроси пользователя
6. Если файл `.v8-project.json` не найден — предложи `/db-list init`

После выполнения: если использованная база не зарегистрирована — предложи добавить через `/db-list add`.

### Автоопределение платформы

Если `v8path` не задан в конфиге:

```powershell
$v8 = Get-ChildItem "C:\Program Files\1cv8\*\bin\1cv8.exe" | Sort-Object -Descending | Select-Object -First 1
```

## Операции

### Инициализировать проект

Запускается после clone, когда `.v8-project.json` отсутствует или неполон.

1. Найди корень проекта рядом с `.git/`.
2. Если есть `.v8-project.json.example`, прочитай его как основу локального
   `.v8-project.json`.
3. Если шаблона нет, обнаружь типовые каталоги:
   - `cfsrc` или `src` → `configSrc`;
   - `extensions` или `ext` → `extensions`;
   - `reports` → `reports`;
   - `processing` или `processors` → `processing`.
4. Определи `v8path` автоматически. Если не удалось — спроси путь к `bin`
   платформы 1С.
5. Спроси параметры хотя бы одной базы: `id`, `name`, `type`, `path` или
   `server/ref`, `user`.
6. Пароль не записывай в `.v8-project.json`. Предложи создать
   `.v8-project.local.json` с паролем и локальными переопределениями.
7. Убедись, что `.v8-project.local.json` и `*.local.json` есть в `.gitignore`
   или предложи добавить эти строки.
8. Создай `.v8-project.json`. Если проекту полезен повторяемый bootstrap для
   команды, предложи создать `.v8-project.json.example` без секретов.

Если `.v8-project.json` уже существует, не перезаписывай его без явного
подтверждения пользователя. Лучше предложи дополнить недостающие поля.

### Показать список баз

Прочитай `.v8-project.json` и применённые local-переопределения, выведи таблицу:

```
ID      Имя           Тип      Путь/Сервер              По умолч.
dev     Разработка    file     C:\Bases\MyApp_Dev       ✓
test    Тестовая      server   srv01/MyApp_Test
```

### Добавить базу

Спроси у пользователя через AskUserQuestion:
- id, name, type (file/server)
- path (для file) или server + ref (для server)
- user (необязательно)
- aliases, branches (необязательно)

Добавь в массив `databases`. Если это первая база — установи как `default`.
Пароль по умолчанию не записывай в `.v8-project.json`: предложи создать или
обновить `.v8-project.local.json`, чтобы секрет не попал в коммит.

### Удалить базу

Удали из массива `databases` по id. Если удаляемая была `default` — спросить новый default.

### Подробности по базе

Выведи все поля конкретной базы.

## Формирование строки подключения

Для использования в шаблонах команд других навыков:

**Файловая база:**
```
/F "<path>"
```

**Серверная база:**
```
/S "<server>/<ref>"
```

**Аутентификация** (добавляется если user задан):
```
/N"<user>" /P"<password>"
```

> **Важно**: между `/N` и именем пробела нет. Между `/P` и паролем пробела нет. Если пароль пустой — опусти `/P` целиком.
